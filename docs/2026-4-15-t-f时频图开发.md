# 2026-4-15 t-f 时频图开发

## 1. 开发目的
- 在不改变时域图主布局与主要交互方式的前提下，新增可交互的短时时频图能力。
- 将原图2、图3整合为 `tab1: 1D Curve`，新增 `tab2: t-f Plot`，支持快速切换分析视图。
- 为 t-f 图增加可调参数（计算模式、尺度、频率范围、颜色映射与颜色范围）。
- 将前面板默认参数集中到单独配置文件，后续改默认值无需再改界面代码。

## 2. UI 与交互改造
### 2.1 右侧绘图区页签化
- 上部保持时域图不变。
- 下部改为分析页签：
  - `1D Curve`：原 Plot2 + Plot3
  - `t-f Plot`：短时时频图 + 颜色条
- 选中页签样式强化：标题文字与背景颜色明显变化。

### 2.2 时域图与 t-f 图联动
- 支持双向 X 轴联动：
  - 时域图缩放/平移 -> t-f 图同步
  - t-f 图缩放/平移 -> 时域图同步
- 新增 X 范围约束逻辑，防止时域图出现越界导致右侧空白区。
- 统一左侧坐标轴宽度，并在 `t-f Plot` 页签激活时给时域图右侧补齐占位，确保上下时间刻度对齐、可视宽度一致。

## 3. t-f 计算与显示
### 3.1 计算模式
- `PSD`：短时功率谱密度。
- `Amplitude`：短时单边振幅谱。
- `Log/Linear` 两种显示尺度可切换。

### 3.2 性能优化（本次重点）
- 旧方案：基于滑窗数组 + 每窗频谱计算，1 分钟 200kHz 数据耗时约 2~3 秒。
- 新方案：在 `processing.py` 中统一改为 `scipy.signal.spectrogram` 向量化路径（`PSD/Amplitude`）。
- 结果：同等参数下，纯计算耗时显著下降（本地测试约 0.1~0.3 秒量级，不含 UI 绘图）。

### 3.3 颜色范围规则
- `t-f color auto` 打开时：
  - `color max` 自动取当前图像数据最大值。
  - `color min` 默认使用 `-120`。
  - 若用户手动改过 `color min`，则自动模式继续沿用用户值。
  - `color min/max` 输入框会回填当前实际采用的刻度值。
- 颜色条背景改为白色。

## 8. 2026-04-16 频率轴映射问题修正

### 8.1 问题现象
- 在时域显示已经启用 `10 kHz ~ 50 kHz` 带通滤波的情况下，`t-f Plot` 中主能量却显示在约 `1.4 ~ 2.2 kHz`
- 经核查，这不是滤波参数传递错误，也不是 `apply_display_filter()` 未被调用
- `t-f Plot` 的输入数据确实来自 `_current_display_values`，即显示滤波后的数据

### 8.2 根因
- `compute_time_frequency_map()` 返回的 `freqs` 是线性间隔的频率 bins
- 原实现中直接计算 `log_freq = np.log10(freqs)`，然后通过一次 `ImageItem.setRect(...)` 把整张谱图铺到对数纵轴上
- 这会产生错误映射：
  - 图像行索引本身仍然对应线性频率顺序
  - 但显示时却被强行解释为等间隔的 `log10(f)` 位置
- `ImageItem` 只能处理仿射意义上的均匀网格，不能直接把“线性频率矩阵”正确显示在“对数频率轴”上

### 8.3 直接后果
- 真实高频能量会被显示到更低的频率位置
- 因此会出现“明明是 10 kHz 以上带通，图上却像是 1~3 kHz 有主能量”的假象

### 8.4 修复方案
- 在 `src/main_window.py` 中新增 `_build_time_frequency_display_grid()`
- 对每个时间列，将 `spectrogram` 的线性频率谱插值到等间距的 `log10(f)` 网格
- 之后再把这份重采样后的矩阵交给 `ImageItem` 显示
- 同时颜色范围计算也改为基于这份实际显示矩阵，保证图像与色条一致

### 8.5 相关代码
- `src/main_window.py::_build_time_frequency_display_grid()`
- `src/main_window.py::_render_time_frequency_image()`
- `src/main_window.py::_apply_time_frequency_color_levels()`
- `src/processing.py::compute_time_frequency_map()`

### 8.6 修复结果
- `t-f Plot` 左侧频率标签与图像能量分布恢复一致
- 对数频率轴下，高频带通结果不再错误显示到低频区
- `10 kHz ~ 50 kHz` 带通后的主能量现在应显示在正确的物理频率位置
