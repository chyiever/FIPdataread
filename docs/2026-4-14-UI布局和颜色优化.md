# 2026-4-14 UI布局和颜色优化

## 背景

本次更新围绕界面可读性、布局层次和一致性交互进行优化，重点解决参数区辨识度不足、控件信息密度过高、标题与边框冲突、以及视觉风格不统一等问题。

## 目标

- 提升左侧参数区的结构清晰度和可识别性
- 统一按钮交互反馈与主功能按钮风格
- 优化颜色体系，保证低饱和、高对比、分层清晰
- 改善顶部品牌展示（标题 + Logo）
- 保持功能逻辑不变，仅优化 UI 呈现与交互体验

## 主要改动

### 1. 整体布局调整

- 新增顶部头部区域：左侧 Logo + 中部标题 `FIP数据处理软件`
- 左侧参数区保持三大块：
  - `Files Management`
  - `File List`
  - `Tab` 区（`Display Controls` / `Short-Time Feature` / `Audio`）
- 左侧参数块之间垂向间距调整为 `30`，增强模块分隔感

### 2. Tab 与参数组织优化

- 将音频相关功能从 `Files Management` 拆分至独立 `Audio` Tab
- 保持音频功能实现不变，仅迁移入口位置：
  - 音频路径
  - 音频降采样
  - Play / Stop / Replay
  - 导出可见音频
- Tab 标题加粗，并增加最小宽度，避免标题显示不全

### 3. 标题与边框可读性优化

- 参数框边框颜色加深，提升三个模块的边界识别
- 解决参数框标题文字与边框重叠问题：
  - 为 `QGroupBox::title` 添加白色背景和内边距

### 4. 颜色与样式体系优化

- 采用低饱和中性色为主色系，蓝色作为交互强调色
- 背景、面板、文本、边框、按钮按明度分层
- 图表区域统一白底 + 低对比网格 + 深浅分层坐标轴
- 按钮默认/悬停/按下/选中/禁用状态全部统一规范

### 5. 交互细节优化

- 所有按钮文字加粗，提升可读性
- 左侧按钮按下态统一为蓝底白字（与右侧模式按钮风格一致）
- `Plot 2` 标签与左侧按钮组增加间距，减少视觉拥挤
- `Visible` 显示精度统一为 3 位小数
- `Visible Window` 输入框默认与精度改为 3 位小数
